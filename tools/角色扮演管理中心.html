📥 安装角色 (导入 ZIP)

把角色 ZIP 放到桌面，然后在 QQ 中向 Bot 发送：

  • /role import — 列出桌面上所有 ZIP
  • /role import 角色名 — 导入指定角色

导入后使用 /role switch 角色名 激活。

💡 Windows → 桌面是 C:\Users\你的用户名\Desktop\
Linux → 桌面是 ~/Desktop/（没有则用 /tmp/）

📤 导出角色

在 QQ 中向 Bot 发送：

  • /role export — 导出当前激活角色到桌面

只打包 config.yaml + audio/ + images/，聊天记录、用户数据绝不导出。

🔄 从服务器同步角色更新

在 QQ 中向 Bot 发送以下命令：

  • /role ping — 测试服务器连通性
  • /role preview — 预览服务器上的音频/图片列表（不下载）
  • /role preview audio — 仅预览音频
  • /role preview images — 仅预览图片
  • /role update — 下载更新的音频+图片（按大小对比，只拉变化的）
  • /role update audio — 仅同步音频
  • /role update images — 仅同步图片
  • /role update status — 本地与远程版本对比

自动按文件大小对比，只下载变化的资源。角色名自动 URL 编码，中文没问题。

如果返回"服务器返回HTML"→ 检查服务器端 Nginx/Caddy 是否把 /api/ 代理到了 Flask。

📤 上传音频/图片到服务器

方式一 — 一键推送（推荐）：

  • /role push — 打包本地全部音频+图片 → ZIP → 上传 → 进入审核队列

一行命令搞定。管理员在后台审核 ZIP 内容后选择上架文件，拒绝恶意文件。

方式二 — 单个上传：

  • /role upload audio expressions/喵.mp3
  • /role upload image mahiro_01.png

路径相对于角色的 audio/ 或 images/ 目录。

📡 配置可信任服务器

在 AstrBot WebUI → 插件设置 → 角色扮演 中填写：

  • trusted_server_url：服务器地址，如 https://kunxun.top
  • trusted_server_token：留空即可 (设备指纹自动认证)

💡 创建/管理角色 → kunxun.top
